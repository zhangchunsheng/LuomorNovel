     LuomorNovel WordPress 主题开发计划

     Context

     用户需要开发一个名为 LuomorNovel 的 WordPress
     区块主题（FSE），用于小说阅读和创作。当前仓库是空的（只有
     README）。主题需要支持：AI
     编写小说（多平台：OpenAI/Claude/Gemini）、小说与章节的
     CRUD、章节目录与排序、按章节阅读、评论、收藏、搜索、分类和标签。

     架构决策

     ┌──────────┬────────────┬─────────────────────────────────────────────────┐
     │   决策   │    选择    │                      理由                       │
     ├──────────┼────────────┼─────────────────────────────────────────────────┤
     │ 主题类型 │ 区块主题   │ WordPress 未来方向，theme.json 集中管理阅读样式 │
     │          │ (FSE)      │                                                 │
     ├──────────┼────────────┼─────────────────────────────────────────────────┤
     │ AI 引擎  │ 多平台抽象 │ OpenAI GPT-4o、Claude Sonnet、Gemini，通过      │
     │          │            │ Settings 配置 Key                               │
     ├──────────┼────────────┼─────────────────────────────────────────────────┤
     │ 数据层   │ 主题内置   │ CPT/分类法/REST API 通过 functions.php          │
     │          │            │ 加载（最佳做法是插件，但用户要求主题）          │
     ├──────────┼────────────┼─────────────────────────────────────────────────┤
     │ 内容关系 │ Post Meta  │ 章节通过 _luomor_novel_id 关联小说，而非层级    │
     │          │            │ CPT                                             │
     ├──────────┼────────────┼─────────────────────────────────────────────────┤
     │ 默认语言 │ 中文 +     │ 界面默认中文，提供 .pot 文件支持翻译            │
     │          │ i18n       │                                                 │
     └──────────┴────────────┴─────────────────────────────────────────────────┘

     数据模型

     - 自定义文章类型: novel（小说容器）+ chapter（章节内容）
     - 分类法: novel_genre（分类/类型，层级）+ novel_tag（标签，扁平）+
     novel_status（状态：连载中/已完结/已暂停）
     - 关键 Meta: _luomor_novel_id（章节→小说关联）,
     _luomor_chapter_order（排序号）, _luomor_ai_provider（AI 提供商）,
     _luomor_ai_prompt（AI 提示词）
     - 用户 Meta: _luomor_bookmarks（收藏的小说 ID 数组）,
     _luomor_last_read_{novel_id}（上次阅读章节）
     - REST 命名空间: luomor/v1

     实施阶段

     Phase 1: 主题基础

     - style.css — 主题头部（WP 识别必需）
     - theme.json — 全局样式：暖色纸感背景、Georgia 衬线阅读字体、720px
     内容宽度、暗色模式
     - functions.php — 引导文件：加载 includes、注册主题支持、挂载资源
     - index.php — 空回退文件
     - 创建目录：includes/, templates/, parts/, assets/css/, assets/js/,
     patterns/, languages/

     Phase 2: 内容结构

     - includes/core.php — 注册 novel/chapter
     CPT、novel_genre/novel_tag/novel_status 分类法、激活时预置状态术语
     - includes/meta.php — 注册自定义 post meta 字段（show_in_rest = true）
     - includes/template-functions.php — 模板辅助函数：获取章节列表（带
     transient 缓存）、获取上下章节、获取阅读进度、获取分类等

     Phase 3: 区块模板

     - templates/home.html — 首页：特色小说轮播、分类导航、搜索
     - templates/single-novel.html — 小说详情：封面、简介、目录（自定义
     luomor/novel-toc 区块）、收藏按钮、阅读进度
     - templates/single-chapter.html — 章节阅读：正文、上下章导航、评论区
     - templates/archive-novel.html — 小说列表：筛选侧栏（分类/状态/排序）、分页
     - templates/taxonomy-novel_genre.html — 分类归档
     - templates/search.html — 搜索结果
     - templates/404.html
     - parts/header.html / parts/footer.html / parts/chapter-nav.html
     - patterns/novel-card.php — 小说卡片模式

     Phase 4: REST API

     - includes/rest-api.php — 注册所有自定义端点：

     ┌────────────────────────────┬───────────────┬───────────────┬────────────┐
     │            端点            │     方法      │     功能      │    权限    │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ novels                     │ GET/POST      │ 列表/创建小说 │ 公开/编辑  │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ novels/<id>                │ GET/PUT/DELET │ 单条/更新/删  │ 公开/所有  │
     │                            │ E             │ 除            │ 者         │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ novels/<id>/chapters       │ GET/POST      │ 章节列表/创建 │ 公开/编辑  │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ chapters/<id>              │ GET/PUT/DELET │ 单条/更新/删  │ 公开/所有  │
     │                            │ E             │ 除            │ 者         │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ novels/<id>/chapters/reord │ POST          │ 批量重排章节  │ 编辑       │
     │ er                         │               │               │            │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ bookmarks/<novel_id>       │ POST/DELETE   │ 收藏/取消     │ 登录       │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ bookmarks                  │ GET           │ 我的收藏列表  │ 登录       │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ reading-progress           │ POST/GET      │ 保存/读取进度 │ 登录       │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ search                     │ GET           │ 搜索小说和章  │ 公开       │
     │                            │               │ 节            │            │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ ai/generate                │ POST          │ AI 生成内容   │ 编辑       │
     ├────────────────────────────┼───────────────┼───────────────┼────────────┤
     │ ai/providers               │ GET           │ 获取可用 AI   │ 公开       │
     │                            │               │ 提供商        │            │
     └────────────────────────────┴───────────────┴───────────────┴────────────┘

     Phase 5: AI 集成

     - includes/ai-service.php — 多提供商抽象层：
       - Luomor_AI_Provider 接口
       - Luomor_AI_OpenAI（chat completions API）
       - Luomor_AI_Claude（messages API）
       - Luomor_AI_Gemini（generateContent API）
       - Luomor_AI_Service 门面类（统一 generate 方法、速率限制、错误处理）
       - 提示词构建器（章节生成、大纲生成、角色生成）
     - includes/settings.php — WP Settings API 主题选项页：
       - AI 配置（各平台 API Key、模型名、默认提供商）
       - 阅读设置（默认字号、暗色模式）
       - 连接测试按钮

     Phase 6: 前端交互

     - assets/js/bookmark.js — 收藏按钮 AJAX 切换
     - assets/js/reading-progress.js — 滚动追踪、自动保存阅读进度
     - assets/js/chapter-sort.js — 管理后台拖拽排序章节
     - assets/js/ai-writer.js — 管理后台 AI
     生成面板（提供商选择、提示词模板、内容插入编辑器）
     - assets/js/search.js — 搜索框实时建议下拉
     - assets/css/reading.css — 阅读样式（字号切换、暗色模式、进度条、章节导航）
     - assets/css/editor.css — 区块编辑器样式
     - assets/css/admin.css — 管理后台样式

     Phase 7: 搜索与分类过滤

     - 增强搜索端点支持 genre/tag/status 过滤
     - 归档页面添加分类/状态筛选 UI
     - 实时搜索建议（search.js）

     Phase 8: 打磨

     - i18n：所有 PHP/JS 字符串国际化，生成 .pot 文件
     - 响应式设计：移动端适配
     - 无障碍：ARIA 标签、焦点管理、色彩对比度
     - 性能：transient 缓存、懒加载、meta 键索引

     关键文件

     ┌─────────────────────────┬──────────────────────────────────────────┐
     │          文件           │                   职责                   │
     ├─────────────────────────┼──────────────────────────────────────────┤
     │ functions.php           │ 引导：模块加载顺序、主题支持、资源挂载   │
     ├─────────────────────────┼──────────────────────────────────────────┤
     │ includes/core.php       │ 数据基础：CPT + 分类法注册               │
     ├─────────────────────────┼──────────────────────────────────────────┤
     │ includes/rest-api.php   │ API 表面：所有 CRUD/收藏/AI 端点         │
     ├─────────────────────────┼──────────────────────────────────────────┤
     │ includes/ai-service.php │ 核心业务逻辑：多平台 AI 抽象             │
     ├─────────────────────────┼──────────────────────────────────────────┤
     │ theme.json              │ 阅读体验定义：字体、色彩、间距、暗色模式 │
     └─────────────────────────┴──────────────────────────────────────────┘

     验证方案

     1. 在本地 WordPress 安装中激活主题
     2. 创建测试小说和章节，验证 CRUD
     3. 配置 AI API Key，测试章节内容生成
     4. 测试章节目录排序和阅读体验
     5. 测试收藏功能（登录/未登录用户）
     6. 测试搜索和分类过滤
     7. 验证响应式布局（移动端/桌面端）
     8. 运行 wp i18n make-pot 验证翻译完整性
